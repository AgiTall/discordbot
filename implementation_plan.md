
# Фаза 2 и 3: Синхронизация, Сохранение данных и Эмодзи

Эта фаза посвящена решению проблемы потери данных (рангов) после перезапуска сервера (коммитов), а также добавлению новых настроек для эмодзи на сайте.

## Анализ проблемы (Почему слетают ранги)

Сейчас бот хранит данные в файлах локальной файловой системы (`data/leveling.db` (SQLite) и `data/economy.db` (JSON)). 
Поскольку бот хостится на Render как "Web Service" (без платного Persistent Disk), **при каждом новом деплое (коммите) Render создает новый чистый контейнер, и все локальные изменения файлов удаляются навсегда**. 
По этой причине всё, что пользователи делают в боте между коммитами (включая настройку рангов, балансы и т.д.), сбрасывается.

## Proposed Changes

### Синхронизация рангов (Сайт <-> Сервер)
- Ранги сейчас устанавливаются командой `/rankrole`.
- Я обновлю веб-панель (`dashboard.html`) так, чтобы она корректно подгружала список установленных рангов из API и позволяла их редактировать прямо на сайте.
- Я переведу таблицу рангов на использование устойчивого хранилища (после вашего ответа про базу данных).

### Настройки Эмодзи (Сайт <-> Сервер)
- В `dashboard.html` я добавлю новую вкладку "Эмодзи".
- В ней будут поля для настройки всех кастомных эмодзи (монеты 🪙, доллары $, карта 🗺️, статистика 👤, графики 📈).
- Эти поля будут синхронизироваться с сервером через существующий API `PUT /api/guilds/<id>/settings` (на сервере они уже поддерживаются в файле `src/guild_config.py`).
- После сохранения бот будет использовать именно те эмодзи, которые вы настроили.
